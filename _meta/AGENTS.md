# AGENTS.md

## Mission

Build a portfolio that convinces hiring managers Ryan is a senior-level design leader and systems thinker who can define strategy, lead complex product work, and help ship real products without relying on buzzwords, trends, or inflated claims.

Every recommendation should improve hiring outcomes.

If a technically correct recommendation would reduce clarity, readability, credibility, or hiring effectiveness, prioritize hiring effectiveness.

# Decision Framework

When making suggestions, prioritize:

1. Clarity
2. Simplicity
3. User understanding
4. Performance
5. Maintainability
6. Visual polish
7. Technical sophistication

Never prioritize novelty.

Never add complexity solely to appear modern.

---

# Technical Philosophy

Use the simplest solution that solves the problem well.

Prefer:

- Semantic HTML
- Accessible markup
- Progressive enhancement
- Vanilla JavaScript
- Minimal dependencies
- Fast-loading pages
- Clear architecture
- Maintainable code

Frameworks may be used when they create measurable value.

Avoid introducing tools solely because they are popular.

---

# Accessibility

Accessibility is a default requirement.

Prioritize:

- Semantic structure
- Keyboard navigation
- Focus visibility
- ARIA when necessary
- Accessible forms
- WCAG compliance
- Reduced motion support

Accessibility should never be treated as optional.

---

# SEO Philosophy

SEO exists to help qualified hiring managers discover the portfolio.

SEO recommendations should improve:

- Search visibility
- Human readability
- Content quality
- Structured data
- Metadata quality

Avoid SEO tactics that reduce readability.

# Case Study Rules

Never invent:

- Metrics
- Revenue impact
- Conversion improvements
- User research findings
- Business outcomes
- Project scope
- Team structure

Reasonable synthesis and interpretation are allowed.

Unsupported claims should be rewritten conservatively.

Always prefer credibility over persuasion.

---

# Visual Design Principles

Inspired by:

- Stripe
- Linear
- Figma
- Vercel
- Apple

Characteristics:

- Strong hierarchy
- Deliberate spacing
- Clear organization
- Minimal visual noise
- Focused interactions

Never introduce visual noise.

Never add decoration without purpose.

Never sacrifice clarity for aesthetics.

---

# UX Writing

Prefer:

- Specific language
- Human language
- Direct language
- Evidence-based language

Avoid:

- Corporate jargon
- Marketing language
- Buzzwords
- Empty claims
- Generic descriptions

Show expertise through substance, not adjectives.

---

# AI Behavior

When uncertain:

- Ask questions before making assumptions.

When proposing improvements:

- Explain rationale briefly.
- Focus on impact.
- Avoid lengthy explanations.

Do not recommend changes merely because they are technically possible.

Recommend only changes that materially improve:
- hiring outcomes
- user experience
- accessibility
- credibility
- maintainability
